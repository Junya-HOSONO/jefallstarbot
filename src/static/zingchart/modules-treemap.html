Treemap Module

Treemap is a visualization module for displaying hierarchical data by using nested rectangles. Treemaps display hierarchical (tree-structured) data as a set of nested rectangles. Each branch of the tree is given a rectangle, which is then tiled with smaller rectangles representing sub-branches. A leaf node's rectangle has an area proportional to a specified dimension on the data. It is also possible to color the leaf nodes to show a separate dimension of the data.

Treemap module in ZingChart uses the children attribute to define the hierarchy and value, value-size and value-aspect atributes to define the data associated with the size of the rectangles and with the aspect.

 

Data Format

  

{
	"graphset" : [
		{
			"type" : "treemap",
			"options" : {
				"max-depth" : ...,
				"max-children" : ...,
				"alpha-aspect" : ...,
				"min-alpha" : ...,
				"split-type" : "...",
				"aspect-type" : "...",
				"box" : {
					...
				},
				"color-start" : "...",
				"color-end" : "...",
				"tooltip-box" : {
					...
				},
				"tooltip-group" : {
					...
				}
			},
			"series" : [
				{
					"text" : "Parent 1",
					"children" : [
						{
							"text" : "Child 1-1",
							"value" : 26
						},
						{
							"text" : "Child 1-2",
							"value" : 63
						},
						...
					]
				},
				{
					"text" : "Parent 2",
					"children" : [
						{
							"text" : "Child 2-1",
							"value" : 13
						},
						{
							"text" : "Child 2-2",
							"children" : [
								{
									"text" : "Child 2-2-1",
									"value" : 32
								},
								{
									"text" : "Child 2-2-2",
									"value" : 85
								},
								...
							]
						},
						...
					]
				},
				...
			]
		}
	]
}

 

JSON Attributes/Syntax

options 

Main object for definitions and styling directives.
  Attribute Type Info
alpha-aspect
alphaAspect		 Boolean

Sets whether alpha will be also applied on the nodes, based on the value-aspect values.

true | false | 1 | 0
aspect-type
aspectType		 String

Sets the aspect used by the treemap.

"random" | "palette" | "red" | "green" | "blue" | "transition"
box Object

Sets the styling for the treemap boxes (nodes).

{...}
color-end
colorEnd		 Mixed

Sets the ending color of the color transition used when aspect-type is set to transition.

"none" | "transparent" | "#f00" | "#f00 #00f" | "red yellow" | "rgb(100, 15, 15)" | ...
color-start
colorStart		 Mixed

Sets the starting color of the color transition used when aspect-type is set to transition.

"none" | "transparent" | "#f00" | "#f00 #00f" | "red yellow" | "rgb(100, 15, 15)" | ...
hover-state
hoverState		 Object

Sets the styling for the hover-state of the treemap boxes (nodes).

{...}
max-children
maxChildren		 Mixed

Sets the maximum number of children each node can have.

4 | [5, 3, 2] | ...
max-depth
maxDepth		 Numeric

Sets the maximum depth for the treemap. Child nodes with levels higher that this setting will be ignored.

5 | 10 | ...
min-alpha
minAlpha		 Numeric

Sets the minimum value of the alpha applied to the nodes, in case alpha-aspect is set to true.

0.3 | 0.9 | ...
palette Array

Defines a custom palette used by the treemap if aspect-type is set to palette. It can hold an array of strings representing colors or an array of objects having various styling attributes like complex background colors, background images or gradient definitions.

["#f90", "#0ff", ...] | [{...}, {...}, ...]
split-type
splitType		 String

Sets the splitting algorithm for the treemap.

"alternate" | "balanced" | "horizontal" | "vertical"
tooltip-box
tooltipBox		 Object

Sets the stying for the tooltip that appears over the final nodes (without children).

{...}
tooltip-group
tooltipGroup		 Object

Sets the stying for the tooltip that appears over the nodes that have at least one child.

{...}
