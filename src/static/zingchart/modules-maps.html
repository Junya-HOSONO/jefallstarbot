Maps Module

The ZingChart library (currently only the HTML5 version) can render various maps via its maps module. In order to render a map at least two modules are required:

  • maps
  • maps-XXX (see available module names below)
Note: the maps and maps-XXX modules are not auto-loaded so loading them must be explicit either by including the respective .js files in the source code or by calling zingchart.loadModules() method: 

zingchart.loadModules('maps,maps-XXX');

 

Available Maps

Map ([JSON name] [module])
World Continents ([world.continents] [maps-world-continents])
World Countries ([world.countries] [maps-world-countries])

Groups: NORTHAMERICA, SOUTHAMERICA, ASIA, EUROPE, AFRICA, AUSTRALIA
USA ([usa] [maps-usa])

 

Data Format

Adding a map requires adding an object in the shapes collection with type attribute set to zingchart.maps. Other map placement and content options will have to be set in the options object. 

{
	graphset : [
		{
			tooltip : {},
			shapes : [
				{
					type : "zingchart.maps",
					options : {
						name : "world.countries",
						id : "world",
						x : 0,
						y : 0,
						width : 640,
						height : 400,
						items : [...],
						ignore : [...],
						groups : [...],
						style : {
							...
							hoverState : {
								...
							},
							tooltip : {
								...
							},
							label : {
								...
							},
							items : {
								ID1 : {
									...
								},
								ID2 : {
									...
								},
								...
							}
						}
					}
				}
			]
		}
	]
}

 

zingchart.maps

zingchart.maps 

Main object for maps module, provides scaling settings and various API's for getting map and map items as well as bi-directional geo API's.
  Attribute Type Info
FORCESCALE Boolean

Specifies whether the mapping engine should accept or not the map aspect given by the map dimensions and the items included in the map. By default, in case the width and the height of the map does not fit the longitude and latitude ratios of the map's bounding box, the user gets a warning icon informing about aspect inconsistency.

true | false | 1 | 0
getInfo Function

Returns various information of the map: placement information, map's latitude and longitude bounding box, content filters (items, ignore and group lists).

zingchart.maps.getInfo(mapid)
getItemInfo Function

Returns various information of a specific item of the map: bounding box, coordinates, label information.

zingchart.maps.getItemInfo(mapid, itemid)
getItems Function

Returns a list of items available in the current map.

zingchart.maps.getItems(mapid)
getLonLat Function

Returns calculated longitude and latitude for a specific x/y point on the map.

zingchart.maps.getLonLat(mapid, xy)
getXY Function

Returns calculated x and y values for a longitude/latitude point on the map.

zingchart.maps.getXY(mapid, lonlat)

 

zingchart.maps API

zingchart.maps.getInfo(mapid) 

Returns various information of the map: placement information, map's latitude and longitude bounding box, content filters (items, ignore and group lists).
  Attribute Type Info
mapid String

Map's id specified in the JSON (if API is called after map was rendered) or the map name (see list above) if the API is being called before map rendering.

"mapid" | ...

 

zingchart.maps.getItemInfo(mapid, itemid) 

Returns various information of a specific item of the map: bounding box, coordinates, label information.
  Attribute Type Info
itemid String

The id of the map item. To see available items getItems() can be called before.

"itemid" | ...
mapid String

Map's id specified in the JSON (if API is called after map was rendered) or the map name (see list above) if the API is being called before map rendering.

"mapid" | ...

 

zingchart.maps.getItems(mapid)

zingchart.maps.getLonLat(mapid, xy) 

Returns calculated longitude and latitude for a specific x/y point on the map.
  Attribute Type Info
mapid String

Map's id specified in the JSON. This API can be called only after map rendering.

"mapid" | ...
xy Array

Array of x and y coordonates relative to the chart's top-left corner.

[200, 100] | ...
