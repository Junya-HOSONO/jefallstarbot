Introduction

Thanks for trying ZingChart. With the ZingChart library, you can render your charts in both HTML5 and Flash to deliver maximum compatibility to your users. This introduction will help you build your first chart and point you in the right direction for further exploration.

 

Requirements

No plugins are necessary to develop HTML5 charts. A modern Web browser with Adobe Flash player 10+ is needed for flash charts and to use the online builder, otherwise no particular IDE or programming environment is required to develop with ZingChart.

Quick Start

  1. Extract package files into a directory

  2. Create a sample file named hellozing.html and include the following code in the <head> tag. zingchart.render is the method call that will call the zingchart library and create the graph in the page. It is typically called in document.ready or window.onload. zingchart.render accepts several possible parameters. 

    <script src="flash_scripts/zingchart-1.1.js"></script> <!-- Flash Library -->
<!-- or -->
<script src="html5_scripts/zingchart-html5-min.js"></script> <!-- HTML5 Library -->
<script>
window.onload = function() {
	zingchart.render({
		id : 'zingchart',
		output : 'canvas', // or use 'flash' or 'svg', Internet Explorer versions 6,7 and 8 will render using VML
		width : 800,
		height : 400,
		dataurl : 'graphfile.txt',
		liburl : 'flash_scripts/zingchart.swf' // in case Flash Library is used
	});
};
</script>

    Note: Your paths may be different depending on where you installed the scripts.

  3. In hellozing.html create a <div> holder for the chart where you want the graph to be displayed.

    <div id="zingchart"></div>

  4. Create the graphfile referenced above called graphfile.txt which will contain JSON for your first chart.

    {
	"graphset" : [
		{
			"type" : "line",
			"series" : [
				{
					"values" : [5, 10, 15, 5, 10, 5]
				}
			]
		}
	]
}

  5. Load hellozing.html in your browser and you're on your way.

 

Chart Output

The ZingChart library can create charts in canvas, svg, vml, and flash. The library implements a rendering system in order to provide you with the appropriate output for any given browser. In order to take advantage of the system, please include both the flash and the HTML5 libraries as well as provide the liburl in the zingchart.render method call. If you do not ever want to use flash, it is fine to omit the flash library and the liburl attribute. If you only want to use flash, you can omit the HTML5 library.

You can set the output attribute in the zingchart.render call if you have a preference to output. This output will always be used unless it cannot render. If it cannot render, ZingChart provides a fallback mechanism:

  1. if output is set to any of the following: auto (which translates to svg for mobile devices and canvas for the rest), canvas, svg, vml and the respective output setting is not supported, then the following fallback cycle is used:

    try: canvas > svg > vml > flash

  2. if output is set to flash and flash is not installed, the following fallback cycle is used:

    try: flash > canvas > svg > vml

 

JSON Inclusion

JSON (JavaScript Object Notation) is used to provide styling and data details to the graph. You can view the API Docs for the various JSON objects and attributes. You can also create customized JSON using the ZingChart Builder.

The JSON object can be included in a variety of ways. If the JSON is built in page, it is set through zingchart.render using the data attribute. If the JSON is included from a separate page, use the dataurl attribute.

Note: When using flash, it is necessary for the JSON to be encoded as a string. When solely using HTML5, it is fine to use the JavaScript object.

  1. Create an inline JavaScript variable

    var jsonConfig = '{' +
	'"graphset" : [' +
		'{' +
			'"type" : "line",' +
			'"series" : [' +
				'{' +
					'"values" : [5, 10, 15, 5, 10, 5]' +
				'}' +
			']' +
		'}' +
	']' +
'}';

    or simply (if you use the HTML5 library):

    var jsonConfig = {
	graphset : [
		{
			type : "line",
			series : [
				{
					values : [5, 10, 15, 5, 10, 5]
				}
			]
		}
	]
};
  2. Create the JSON object through the server-side and output it into a JavaScript variable

    <?php
$values = "5, 10, 15, 5, 10, 5";
$json = '{
	"graphset" : [
		{
			"type" : "line",
			"series" : [
				{
				"values" : [' . $values . ']
				}
			]
  		}
	]
}';
$json = str_replace(array("\r","\n"), "", $json);
?>
<script type="text/javascript">
var jsonConfig = '<?php echo $json; ?>';
</script>
  3. Create a static text file containing a valid JSON structure

    {
	"graphset" : [
		{
			"type" : "line",
			"series" : [
				{
				"values" : [5, 10, 15, 5, 10, 5]
				}
			]
		}
	]
}
  4. Create a server-side script that creates a valid JSON structure

    <?php
$values = "5, 10, 15, 5, 10, 5";
$json = '{
	"graphset" : [
		{
			"type" : "line",
			"series" : [
				{
					"values" : [' . $values . ']
				}
			]
		}
	]
}';
$json = str_replace(array("\r","\n"), "", $json);
echo $json;
?>

Submitting Bugs

As with any product, you may occasionally discover a bug. If so, don't panic; submitting bugs with ZingChart is quick and easy. Of course if you're using the free trial you can panic a little bit, because only licensed developers get full priority support. (Sorry we gotta eat too) Here are a few ways to notify us of bugs:

In Graph Error Reporting

The easiest way to submit a bug report is to utilize the feature built-in to ZingChart itself. On any active chart right-click to bring up a context menu and select Submit bug.



A dialog to populate will allow you to submit a description of your bug, the current config used to construct your graph and contact information if you desire follow-up.



We will also collect your Flash version, ZingChart build number, and basic configuration information. By default we will capture the image of the graph you are currently submitting a bug for. If for some reason you do not want to disclose this information you may deselect these choices. Be aware, that bug retification will be severly hampered if you do not submit a full request.

After hitting the submit button the data will be collected and sent to the ZingChart support team for evaluation.

Submit a Request Online

You may submit a bug or feature request using our standard support form on the ZingChart website.

Email Us

You may submit a bug or feature request by emailing us at support@zingchart.com.

Tweet Us

While we'd love you to show us love on twitter, you can also request help or share ideas by mentioning @ZingChart or including the #zingchart hashtag in your tweets. Regardless, follow us for cool updates, charts, and other nonsense.
