ZingChart Documentation

 

Welcome to the ZingChart documentation tool. Our goal is to make implementing ZingChart as easy and intuitive as possible (note the CSS-like attribute naming). Here you'll find information on getting started, chart rendering features, chart styling and customization, API methods (for chart updating/modifying), API events, and links to other resources.

Introduction

This simple tutorial walks you thru your first chart with ZingChart and introduces important concepts such as the different ways to include ZingChart in a page, the wide range of render options, and the general structure of ZingChart JSON packet.

«zingchart» Object

The zingchart object contains properties to set global attributes such as general font face or size information. The object also contains two important methods: zingchart.render() which controls rendering formats and characteristics, and zingchart.exec() which provides a mechanism to invoke various API calls on a chart.

JSON Syntax

Charts in ZingChart are primarily defined using JSON syntax. ZingChart contains a rich set of options for configuring all aspects of charts including: plot data, scales, callouts, legends, title, interactive elements and numerous visual characteristics.

API Methods

ZingChart provides a rich API for controlling chart characteristics from re-rendering a plot (reload), loading new data (setseriesdata), modifying existing charts (modifyplot), zooming in on a data range (zoomto), toggling various interactive features (togglelegend), and much more.

API Events

The ZingChart library raises several events that can be captured by your JavaScript scripts. The event model is quite granular handling node level events (node_click, node_mouseover, node_mouseout) to chart wide events like interacting with the legend, hiding and showing plots, zooming, and dozens more.

Examples

Sometimes the best way to learn chart building is by seeing some demos and working backwards. We provide a number of demos for you in the package and many more online. Always remember that to see the JSON syntax to right click and "View Source".
