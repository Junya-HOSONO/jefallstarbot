ZingChart Modules

The ZingChart library allows customers to include/use only specific sections of the library in order to minimize the size of the included files.

 

Including Modules

ZingChart modules can be included using various approaches:

  1. Inserting manually the js files in the HTML code: 

    <head>
	...
	<script src="zingchart-html5-core-min.js"></script>
	<script src="zingchart-html5-xy-min.js"></script>
	<script src="zingchart-html5-line-min.js"></script>
	<script src="zingchart-html5-tooltip-min.js"></script>
	...
</head>

  2. Using the auto-loading capability of the ZingChart library and the zingchart.loadModules API:

    The core module of the ZingChart library has the ability to scan the data JSON and to auto-load the required modules. This is valid for most of the modules, that can be detected through the JSON. The other modules have to be explicitely loaded. 

    {
		"type":"line",
		"tooltip":{},
		"crosshair-x":{},
		"scale-x":{
			"zooming":true
		},
		"series":[
			{
				"values":[1,3,4]
			}
		]
	}
    <head>
		...
		<script src="zingchart-html5-core-min.js"></script>
		<script>
			zingchart.MODULESDIR = '/path/to/modules';
			zingchart.loadModule('api,debug');
			zingchart.render({
				...
			});
		</script>
		...
	</head>

    Based on the above JSON, the following modules will be automatically loaded: xy, line, tooltip, guide, zoom. In order to use the API and the debug tools (bug report, view source, error reporting) those modules have to be explicitely added. The folder in which the module loader is looking is specified by zingchart.MODULESDIR variable. By default, the value is ./modules/.

  3. Creating a custom build using the Build Generator.
